Computing the weighted h-index


A common algorithm problem is that given a sequence of numbers, find a h-index. Where h-index is the largest integer hhh such there are at least hhh integers in the sequence is at least as large as hhh.

Formally, we have the following problem.

Problem1

Given a1,…,ana_1,\ldots,a_na1​,…,an​, find the largest hhh, such that ∣{i∣ai≥h}∣≥h|\set{i \mid a_i\geq h}|\geq h∣{i∣ai​≥h}∣≥h.

The h-index problem is featured in leetcode.

If we the numbers are sorted, then a trivial O(n)O(n)O(n) time algorithm exists. If it is not sorted, then note that we can solve the problem on min⁡(a1,n),…,min⁡(an,n)\min(a_1,n),\ldots,\min(a_n,n)min(a1​,n),…,min(an​,n). In this case, the input numbers is at most nnn, therefore can be sorted in O(n)O(n)O(n) time. Hence the total running time is O(n)O(n)O(n).

Consider a weighted version of the problem where the above algorithm does not work.

Problem2

Given a sequence of pairs of non-negative positive reals (w1,a1),…,(wn,an)(w_1,a_1),\ldots,(w_n,a_n)(w1​,a1​),…,(wn​,an​). Find the largest h∈Rh\in \Rh∈R, such that ∑i:ai≥hwi≥h\sum_{i:a_i\geq h} w_i \geq h∑i:ai​≥h​wi​≥h.

An O(n)O(n)O(n) time algorithm still exists. For simplicity, we assume all aia_iai​’s are distinct, so the input is a set. The case where aia_iai​’s are not distinct is left as an exercise to the reader.

Define f(t)=∑i:ai≥twif(t) = \sum_{i:a_i\geq t} w_if(t)=∑i:ai​≥t​wi​. We want to find the largest ttt such that f(t)≥tf(t)\geq tf(t)≥t. First, we can find the median of a1,…,ana_1,\ldots,a_na1​,…,an​, say ttt. If f(t)<tf(t) < tf(t)<t, then we recurse on {(wi−f(t),ai)∣ai<t}\set{(w_i-f(t),a_i) \mid a_i< t}{(wi​−f(t),ai​)∣ai​<t}. Assume the optimum in the recursed solution is t′t't′, we return t′+f(t)t'+f(t)t′+f(t) as the solution. If f(t)≥tf(t)\geq tf(t)≥t, then we recurse and output the solution with input {(wi,ai)∣ai≥t}\set{(w_i,a_i) \mid a_i\geq t}{(wi​,ai​)∣ai​≥t}. The running time satisfies T(n)=T(n/2)+O(n)T(n)=T(n/2)+O(n)T(n)=T(n/2)+O(n), which is O(n)O(n)O(n).
