Word break with cost


Problem1

Given a set of strings WWW, a cost function c:W→Rc:W\to \Rc:W→R and a string sss. Find elements w1,…,wk∈Ww_1,\ldots,w_k\in Ww1​,…,wk​∈W such that s=w1…wks=w_1\ldots w_ks=w1​…wk​, and ∑i=1kc(wi)\sum_{i=1}^k c(w_i)∑i=1k​c(wi​) is minimized.

This problem is a generalization of the word break problem on leetcode. Many algorithms you see online assumes that string in WWW has constant length, checking the hash table takes O(1)O(1)O(1) time, and obtain an O(n2)O(n^2)O(n2) time algorithm. It is not as easy. Here we show an algorithm that considers the strings in WWW have arbitrary length.

Consider the following graph G=(V,E)G=(V,E)G=(V,E), where V={0,…,n}V=\set{0,\ldots,n}V={0,…,n}, and there is an edge from iii and jjj, if s[i+1..j]=w∈Ws[i+1..j]=w\in Ws[i+1..j]=w∈W, and the label of the edge (i,j)(i,j)(i,j) is the string www, and the cost is c(w)c(w)c(w). Let zzz be number of substrings in sss matches some element in WWW. The graph has zzz edges. Note z=O(nL)z=O(n\sqrt{L})z=O(nL​). Indeed, the sum of the length of the labels of all outgoing edges cannot be more than LLL, and the length of each label is different. Hence each vertex can have at most O(L)O(\sqrt{L})O(L​) outgoing edges. The graph is a DAG, so we can find the shortest path from 000 to nnn in linear time with respect to the number of edges. This shows if we can compute the graph in O(z+L)O(z+L)O(z+L) time, then we solve the problem in O(z+L)O(z+L)O(z+L) time.

We can build the Aho–Corasick automaton for WWW in O(L)O(L)O(L) time. It can be used to find all substrings of sss that matches something in WWW by traversing the automaton once. The running time is the total number of substrings matched, which is O(z)O(z)O(z). Hence building the graph takes O(z+L)O(z+L)O(z+L) time. zzz is clearly no more than nmnmnm, where m=∣W∣m=|W|m=∣W∣. Also, it is also clear z=O(nL)z=O(n\sqrt{L})z=O(nL​). Indeed, there can be at most O(L)O(\sqrt{L})O(L​) edges start from iii, since each edge has a label of different length, and sum of those length labels is no larger than LLL.

If we only want to know if there exists a solution, then there is a O~(nL1/3+L)\tilde{O}(nL^{1/3}+L)O~(nL1/3+L) time algorithm [1]. The algorithm is close to optimal assuming the algorithm is combinatorial and the alphabet can be arbitrarily large.

Can we obtain similar running time for the word break with cost problem? There are evidence against it. If the alphabet is unary, this problem is equivalent to the unbounded knapsack problem, which likely does not have an algorithm with running time O((n+L)2−ε)O((n+L)^{2-\e})O((n+L)2−ε) for any ε>0\e>0ε>0 [2] and mmm can be as large as Ω(L)\Omega(\sqrt{L})Ω(L​). Of course, this does not mean there might not be a O(nL1/3+L)O(nL^{1/3}+L)O(nL1/3+L) time algorithm, since the reduction involved in the paper might not hold when we require m=Ω(L)m=\Omega(\sqrt{L})m=Ω(L​).
