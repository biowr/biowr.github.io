Bottleneck kkk-link path


A DAG is called complete, if there are vertices v1,…,vnv_1,\ldots,v_nv1​,…,vn​, and vivjv_iv_jvi​vj​ is an edge if and only if i<ji<ji<j. Let w(i,j)w(i,j)w(i,j) be the edge weights from iii to jjj. The weight is called ordered, if w(i,j)<w(i,j+1)w(i,j)<w(i,j+1)w(i,j)<w(i,j+1) and w(i+1,j)<w(i,j)w(i+ 1,j)<w(i,j)w(i+1,j)<w(i,j).

Problem1Bottleneck kkk-link path problem

Find a path consists of kkk edges from v1v_1v1​ to vnv_nvn​, such that the maximum weight of the edges in the path is minimized.

One can formulate a dynamic programming algorithm, which takes O(kn2)O(kn^2)O(kn2) time. My previous writing shows an O(kn)O(kn)O(kn) time algorithm using the monge property. Using binary search, there is also an O(klog⁡(n/k)log⁡M)O(k\log(n/k)\log M)O(klog(n/k)logM) time algorithm if all weights are positive integers no larger than MMM.

We show there is an O(n+klog⁡(n/k)log⁡n)O(n+k\log(n/k)\log n)O(n+klog(n/k)logn) time algorithm. Assume λ∗\lambda^*λ∗ is the optimal weight. First, there is an oracle that given λ\lambdaλ, decides if λ≥λ∗\lambda\geq \lambda^*λ≥λ∗. Indeed, we can apply the greedy algorithm. Find the sequence a1,…,aka_1,\ldots,a_ka1​,…,ak​, as follows. a0=1a_0=1a0​=1, aia_iai​ is the largest value such that w(ai−1,ai)≤λw(a_{i-1},a_i)\leq \lambdaw(ai−1​,ai​)≤λ. If ak=na_k=nak​=n, then it is clear that λ≥λ∗\lambda \geq \lambda^*λ≥λ∗. Also, we can show if ak<na_k<nak​<n, then λ<λ∗\lambda < \lambda^*λ<λ∗. O(n)O(n)O(n) time seems to be a quite large bound. We could do it in O(klog⁡(n))O(k\log (n))O(klog(n)) instead by doing binary search for each aia_iai​. Using exponential search instead, we can obtain a O(klog⁡(n/k))O(k\log(n/k))O(klog(n/k)) time algorithm.

One need to do binary search for λ∗\lambda^*λ∗. There are Ω(n2)\Omega(n^2)Ω(n2) weights, let it be WWW. One does not have to know all of them in order to apply binary search. Note that www is a matrix sorted in both row and column, hence we need a selection algorithm that returns the kkkth smallest element for such matrix. There is an O(n)O(n)O(n) time algorithm for that. Hence we can do binary search on the sorted WWW by spending O(n)O(n)O(n) time to access iiith element. We now obtain a O((n+klog⁡(n/k))log⁡n)=O(nlog⁡n)O((n+ k\log(n/k)) \log n) = O(n\log n)O((n+klog(n/k))logn)=O(nlogn) time algorithm. Not bad.

We can speed it up even further. Instead of selection in the sorted matrix, we can do search in the sorted matrix. We are given an oracle to test if a value is smaller than λ∗\lambda^*λ∗ after all. We can do search for λ∗\lambda^*λ∗ using O(log⁡n)O(\log n)O(logn) oracle calls and O(n)O(n)O(n) time. Hence this gives us a O(n+klog⁡(n/k)log⁡n)O(n+k\log (n/k) \log n)O(n+klog(n/k)logn) time algorithm for the problem. Whenever k=O(nlog⁡(n)log⁡log⁡n)k=O(\frac{n}{\log(n) \log \log n})k=O(log(n)loglognn​), this is O(n)O(n)O(n) time.

As an application, we obtain a solution to Leetcode 410 Split Array Largest Sum. The problem is also called the linear partitioning problem. The problem asks one to partition array into kkk contagious subarrays that minimizes the maximum sum of each subarray. It was an example for learning dynamic programming in chapter 8.5 of [1]. An O(kn2)O(kn^2)O(kn2) algorithm was given. Reading the discussion online, one would find O(nlog⁡M)O(n\log M)O(nlogM) time algorithm is the suggested solution, where MMM is the maximum over all integers. The algorithm is actually fairly useful for photo galleries. There is the NPM package linear-partitioning, used by multiple photo galleries packages. My first encountered of the problem was also for photo gallery. The linear partition problem reduces to the bottleneck kkk-link path problem because we can define w(i,j)w(i,j)w(i,j) to be the sum of elements from the iiith index of the array to the jjjth index of the array. After O(n)O(n)O(n) preprocessing, w(i,j)w(i,j)w(i,j) can be computed in O(1)O(1)O(1) time. This results a O(n+klog⁡(n/k)log⁡n)O(n+k\log (n/k) \log n)O(n+klog(n/k)logn) running time algorithm.

What about when kkk is large? I’ve emailed Samson Zhou, who has confirmed the algorithm in [2] can be used to solve the problem in O(n)O(n)O(n) time.
