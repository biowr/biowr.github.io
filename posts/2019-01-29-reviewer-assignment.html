A Reviewer Assignment Problem


Consider there are some reviewers and some papers. Each reviewer can review exactly one paper, and each reviewer is qualified to review some subset of papers. We are interested in maximizing the number of papers reviewed by at least kkk reviewer, then under that constraint, maximize the paper reviewed by k+1k+1k+1 reviewer, etc. This make sure we are being fair in evaluating papers. It would try to avoid the case where most paper getting small number of reviews and a few papers getting unreasonable number of reviews.

When k=3k=3k=3, the problem is NP-hard, since it would imply we have to maximized gM(3)g_M(3)gM​(3), and this is already NP-hard because exact cover by 333-sets. That is, given a collection of sets of size 333 each. Decide if there exists a subcollection that forms a partition of the universe.

1 Extensions

I first heard of the problem from [3], where they focused on k=1k=1k=1 case, but the reviewer can review more than one paper. This case can be handled easily. We can add degree upper and lower bounds to all vertices, and only look at subgraphs in MMM that satisfies the upper and lower bounds. That is, we can also make sure no reviewers review too many papers too. Under that constraint, find (gM(k),…,gM(n))(g_M(k),\ldots,g_M(n))(gM​(k),…,gM​(n)) lexicographically. This is possible but more tricky, as we have to do some reduction from capacitated bbb-matching to bbb-matching.

There is a little more generalization. Assume for each paper, we have a lower bound of reviews dvd_vdv​. That is, it has to be reviewed by at least dvd_vdv​ person to be useful. So translating to the graph case, we can impose the constraint that deg⁡M(v)=0\deg_M(v)=0degM​(v)=0 or deg⁡M(v)≥dv\deg_M(v)\geq d_vdegM​(v)≥dv​. One can see maximizing (gM(2),gM(3),…,gM(n))(g_M(2),g_M(3),\ldots,g_M(n))(gM​(2),gM​(3),…,gM​(n)) is equivalent to maximizing (gM(1),gM(2),…,gM(n))(g_M(1),g_M(2),\ldots,g_M(n))(gM​(1),gM​(2),…,gM​(n)) where dv=2d_v=2dv​=2 for all vertices. Again, one can modify the reduction to handle the case when dvd_vdv​ is either 111 or 222.
