L1L_1L1​ linear regression


I read an article on the errors in visualization. The example of forcing a relationship by cherry-picking scales is delightful. I recommend reading it.

I am interested in misleading people while being completely honest. The article inspires the following problem. Given 2 vectors x,y∈Rn\bm{x},\bm{y}\in \R^nx,y∈Rn. Let 1\bm{1}1 be the all 111 vector in Rn\R^nRn. We are interested in finding a,b∈Ra,b\in \Ra,b∈R, such that ∥y−(ax+b1)∥p\|\bm{y}-(a\bm{x}+b\bm{1})\|_p∥y−(ax+b1)∥p​ is minimized. Here ppp is either 1,21,21,2 or ∞\infty∞.

Note the problem is precisely the same as the linear regression problem. In the linear regression problem, we are given a point set S⊂R2S\subset \R^2S⊂R2 of size nnn and we are interested in find a line f(x)=ax+bf(x) = ax+bf(x)=ax+b, such that it minimizes the error, defined as

∑(x,y)∈S∥y−f(x)∥p \sum_{(x,y)\in S} \|y - f(x)\|_p (x,y)∈S∑​∥y−f(x)∥p​

For p=2p=2p=2, there is a O(n)O(n)O(n) time algorithm because there is a closed formula. For p=∞p=\inftyp=∞, the problem can be rewritten as a linear program with 333 variables and nnn constraints. Using Megiddo’s result [1], there is a O(n)O(n)O(n) time algorithm to solve this problem.

It is hard to find the worst case complexity when p=1p=1p=1. This case is called the least absolute deviations. Statisticians just don’t care about worst case running time as CS people do.

There are a few methods I found. One is to write it as a linear program on n+2n+2n+2 variables and nnn constraints and solve it using the simplex method. The linear program is as follows.

min⁡a,b,t1,…,tn∑i=1ntis.t.ti≥(axi+b)−yi∀1≤i≤nti≤yi−(axi+b)∀1≤i≤n \begin{aligned} & \min_{a,b,t_1,\ldots,t_n} & & \sum_{i=1}^n t_i & \\ & \text{s.t.} & & t_i \geq (ax_i+b)-y_i & \forall 1 \leq i \leq n \\ & & & t_i \leq y_i-(ax_i+b) & \forall 1 \leq i \leq n \\ \end{aligned} ​a,b,t1​,…,tn​min​s.t.​​i=1∑n​ti​ti​≥(axi​+b)−yi​ti​≤yi​−(axi​+b)​∀1≤i≤n∀1≤i≤n​

There are a bunch of other algorithms that specializes the simplex algorithm on this particular problem. There are also some iterative methods. Unfortunately, those algorithms depends on the actual numbers in the input. I want a running time that only depends on nnn.

There exists an optimal solution that contains two points in SSS. The native algorithm is to try all possible O(n2)O(n^2)O(n2) lines. For each line, the algorithm can compute the error in O(n)O(n)O(n) time. The naive algorithm’s running time is O(n3)O(n^3)O(n3). There is a smarter algorithm. The optimal line that contains the point can actually be found in O(n)O(n)O(n) time. Indeed, consider the line passes through the point (x,y)(x,y)(x,y). We consider changing the slope of the line, while maintaining it still contain (x,y)(x,y)(x,y). One can see a minimum will be reached at some line. Indeed, assume we reorder the points, so yi−yxi−x≤yi+1−yxi+1−x\frac{y_i-y}{x_i-x}\leq \frac{y_{i+1}-y}{x_{i+1}-x}xi​−xyi​−y​≤xi+1​−xyi+1​−y​ (namely, increasing slope). Let kkk be the smallest integer such that the sum of ∑i=1k∣xi−x∣≥∑i=k+1n∣xi−x∣\sum_{i=1}^k |x_i-x|\geq \sum_{i=k+1}^n |x_i-x|∑i=1k​∣xi​−x∣≥∑i=k+1n​∣xi​−x∣. The line determined by (x,y)(x,y)(x,y) and (xk,yk)(x_k,y_k)(xk​,yk​) is the desired line. This can be computed in linear time by finding weighted median. Hence one can show the running time is O(n2)O(n^2)O(n2). This is the idea of [2]. That is all I can find through an hour of search.

After discussing with Qizheng He, he suggested the following approach. Consider the function gp(s)g_p(s)gp​(s) for p∈Sp\in Sp∈S. It is defined as the error for the line of slope sss that contains ppp. The function is bitonic, therefore we can do a ternary search to find the minimum. There are only n−1n-1n−1 possible slopes, hence the ternary search will take O(log⁡n)O(\log n)O(logn) queries, where each query asks for the error of the line that goes through ppp and some other point.

Given a line f(x)=ax+bf(x)=ax+bf(x)=ax+b, can one compute the error quickly? It is possible to decompose it to few halfspace range counting queries (allowing weights). In halfspace counting queries problem, we are given nnn points with weights, we can preprocess it and obtain a data structure. Each query to a data structure is a halfspace, the output is the sum of all elements in the halfspace. In 222D, there exists a preprocessing time O~(n4/3)\tilde{O}(n^{4/3})O~(n4/3) and query time O~(n1/3)\tilde{O}(n^{1/3})O~(n1/3) data structure [3]. Let S+S^+S+ be the set of points above fff, and S−S^-S− be the set of points below fff. The result is precisely the following.

∑(x,y)∈S+y−ax−b+∑(x,y)∈S−ax+b−y \sum_{(x,y)\in S^+} y - ax-b + \sum_{(x,y)\in S^-} ax+b - y (x,y)∈S+∑​y−ax−b+(x,y)∈S−∑​ax+b−y

Let’s consider the second sum, ∑(x,y)∈S−ax+b−y=a∑(x,y)∈S−x+∣S−∣b−∑(x,y)∈S−y\sum_{(x,y)\in S^-} ax+b - y = a\sum_{(x,y)\in S^-}x + |S^-|b -\sum_{(x,y)\in S^-}y∑(x,y)∈S−​ax+b−y=a∑(x,y)∈S−​x+∣S−∣b−∑(x,y)∈S−​y. Note the 333 terms can each be solved with a halfspace counting query, consider all points lies below fff. This shows in 666 halfspace counting queries.

How can one do ternary search? This would need us to be able to pick the point that gives us the iiith largest slope with ppp. We need a data structure such that it can return the iiith largest point in the radial ordering of the points in SSS around ppp. It is equivalent to halfspace range counting up to polylog factors.

Thus, the total running time after building the data structure in O~(n4/3)\tilde{O}(n^{4/3})O~(n4/3) is nnn times ternary search over nnn elements, where each decision process takes O~(n1/3)\tilde{O}(n^{1/3})O~(n1/3) time. Therefore the final running time is O~(n4/3)\tilde{O}(n^{4/3})O~(n4/3) time.

Qizheng mentioned the problem to Timothy Chan, who gave us some references. There is an easy solution that obtains O(nlog⁡2n)O(n\log^2 n)O(nlog2n) time algorithm using simple parametric search [4]. Consider the following linear program. Let kkk be a constant. We are given a1,…,ak,b1,…,bna_1,\ldots,a_k,b_1,\ldots,b_na1​,…,ak​,b1​,…,bn​, kkkD vectors β1,…,βm\beta_1,\ldots,\beta_mβ1​,…,βm​ and reals α1,…,αm\alpha_1,\ldots,\alpha_mα1​,…,αm​. Sets J1,…,JnJ_1,\ldots,J_nJ1​,…,Jn​ a partition of [m][m][m].

min⁡w1,…,wk,x1,…,xn∑i=1kaiwi+∑i=1nbixis.t.xi≥(∑d=1kβj,dwd)−αj∀1≤i≤n,j∈Ji \begin{aligned} & \min_{w_1,\ldots,w_k,x_1,\ldots,x_n} & & \sum_{i=1}^k a_iw_i + \sum_{i=1}^n b_ix_i & \\ & \text{s.t.} & & x_i \geq (\sum_{d=1}^k \beta_{j,d} w_d) - \alpha_j & \forall 1 \leq i \leq n, j\in J_i \end{aligned} ​w1​,…,wk​,x1​,…,xn​min​s.t.​​i=1∑k​ai​wi​+i=1∑n​bi​xi​xi​≥(d=1∑k​βj,d​wd​)−αj​​∀1≤i≤n,j∈Ji​​

Zemel showed such linear program can be solved in O(m)O(m)O(m) time for constant kkk [5]. The idea is a similar algorithm to Megiddo’s linear time constant dimension LP algorithm [1]. For linear regression problem in L1L_1L1​ with nnn data points. The linear program we derived is a special case of the above linear program when k=2k=2k=2 and m=O(n)m=O(n)m=O(n). In fact, Zemel use the same linear program to show constant dimension L1L_1L1​ regression can be solved in linear time.

1 Open problem

One can also define another metric, the lexicographical minimum. Such idea was already present in fairness related linear regression [6]. Once we sort the values of ∣y−f(x)∣|y - f(x)|∣y−f(x)∣ for (x,y)∈S(x,y)\in S(x,y)∈S, say obtaining a1,…,ana_1,\ldots,a_na1​,…,an​, where a1≥a2≥…≥ana_1\geq a_2 \geq \ldots \geq a_na1​≥a2​≥…≥an​. We are interested in finding a fff that minimizes the sequence a1,…,ana_1,\ldots,a_na1​,…,an​, lexicographically. Can this problem be solved in O(n)O(n)O(n) time?
