Find the period of a nice eventually periodic sequence


A sequence is periodic if si=si+ps_i = s_{i+p}si​=si+p​ for all iii, where p>0p>0p>0 is called a period. A sequence is eventually periodic if there exists a nnn and ppp, such that si=si+ps_i = s_{i+p}si​=si+p​ for all i>ni>ni>n. The sequence with index above nnn is the periodic part.

A sequence is called uuu-normal, if there exists si=si+ps_i=s_{i+p}si​=si+p​ for some p>0p>0p>0 for all iii in a interval of length uuu, then the sequence starting at sis_isi​ is part of the periodic part.

When does uuu-normal sequence comes up? Consider we have a recurrence relation that produces a sequence. Say it is of the form an=f(an−1,an−2,…,an−u)a_n = f(a_{n-1},a_{n-2},\ldots,a_{n-u})an​=f(an−1​,an−2​,…,an−u​). The sequence a1,…a_1,\ldotsa1​,… is uuu-normal.

Problem1

Given a oracle that can take input iii and return the iiith element in a uuu-normal eventually periodic sequence aaa. Find the smallest lexicographic pair (n,p)(n,p)(n,p) where p>0p>0p>0, such that ai=ai+pa_i=a_{i+p}ai​=ai+p​ for all i>ni>ni>n.

One can solve this problem in O(ulog⁡nu)O(u \log \frac{n}{u})O(ulogun​) time. First, consider the subsequence au,a2u,…a_u,a_{2u},\ldotsau​,a2u​,…. We guess an upper bound on nnn through exponential search in the subsequence. There is a O(u)O(u)O(u) time algorithm to decide if n′≥nn'\geq nn′≥n. For example using the KMP algorithm. We can quickly locate a n′n'n′ such that n∈[n′−u,n′]n\in [n'-u,n']n∈[n′−u,n′].

Let’s take the sequence an′−u,…,an′+3ua_{n'-u},\ldots,a_{n'+3u}an′−u​,…,an′+3u​. We just need to solve the following problem. Given a O(u)O(u)O(u) length string, find the longest suffix that appears at least twice in the sequence. Let such suffix be sss. We know ∣s∣≥3u|s|\geq 3u∣s∣≥3u, which sss has to overlap with any other occurrence of the sequence. The claim is that the partial match table in the KMP algorithm would give us such information. Hence we can obtain nnn. ppp can also be obtained in the same time. Note that KMP algorithm only uses the fact one can check equality of two elements. So the sequence can contain elements from very general space.

The total running time is O(log⁡nu)O(\log \frac{n}{u})O(logun​) calls to O(u)O(u)O(u) time string matching. The total running time is therefore O(ulog⁡nu)O(u\log \frac{n}{u})O(ulogun​).

In many applications, we do not get oracle access to iiith index of the sequence. But we can read the sequence from the beginning as a list. In that case, we don’t do binary search, but linear search. Advance the index by uuu and obtain n′n'n′, and test if nnn is no larger than the current point. If so, again we have n∈[n′−u,n′]n\in [n'-u,n']n∈[n′−u,n′] and reduce to the previous problem. If not, advance the index by uuu again and repeat. This gives us a O(n+u)O(n+u)O(n+u) time algorithm.
