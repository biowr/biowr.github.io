List the smallest kkk subset sums


Problem1

Given a set of positive reals {x1,…,xn}\set{x_1,\ldots,x_n}{x1​,…,xn​} where x1<x2<…<xnx_1<x_2<\ldots<x_nx1​<x2​<…<xn​, find the smallest kkk subset sums.

We can assume n≤kn\leq kn≤k, because we do not have to read xjx_jxj​ if j>kj>kj>k.

Torsten Gross and Nils Blüthgen posted a O(k2)O(k^2)O(k2) time solution on arXiv.

We show a O(klog⁡k)O(k\log k)O(klogk) time algorithm, which is optimal if we want to output the numbers in order.

We list the sums one by one by maintaining a priority queue of sums. We start with the empty set. Assume that we added the sum induced by I⊂[n]I\subset [n]I⊂[n] (that is, ∑i∈Ixi\sum_{i\in I} x_i∑i∈I​xi​) into the output, let j=1+max⁡Ij=1+\max Ij=1+maxI. Now we can consider two possibilities by extending the current solution: the sum induced by I∪{j}I\cup \set{j}I∪{j} or the sum induced by I∪{k}I\cup \set{k}I∪{k} where k>jk>jk>j. We will add both possibilities to the queue so that one can inspect them later. We can avoid storing the sets, only the values are required.

Here is a python implementation.

def first_k_subset_sums(x,k):
    n = len(x)
    h = []
    output = [0] # need to account for the empty set
    heapq.heappush(h,(x[0],0))
    while h and len(output)<k:
        (u,b) = heapq.heappop(h)
        output.append(u)
        if b+1<n:
            heapq.heappush(h,(u+x[b+1],b+1))
            heapq.heappush(h,((u-x[b])+x[b+1],b+1))
    return output

If we want to output the sets themselves, not just the values, does the running time change? If a set III is in the output, then all subsets of III must also be in the output. Hence the largest set we can ever output has size O(log⁡k)O(\log k)O(logk). Therefore the total output length is at most O(klog⁡k)O(k\log k)O(klogk).

This is also a lower bound. Consider when xi=2ix_i=2^ixi​=2i, then we will output all subsets of {x1,…,xlog⁡k}\set{x_1,\ldots,x_{\log k}}{x1​,…,xlogk​}, and we know that ∑i=1log⁡ki(log⁡ki)=Ω(log⁡k)\sum_{i=1}^{\log k} i{\log k\choose i} = \Omega(\log k)∑i=1logk​i(ilogk​)=Ω(logk).

If we don’t have to list the smallest kkk subset sum values in order, then O(k)O(k)O(k) is possible, see this mathoverflow answer by David Eppstein.

If we are interested in the smallest kkk distinct subset sum. I don’t know of any algorithm that performs better than O(nk)O(nk)O(nk), even if we know that n=Ω(k)n=\Omega(k)n=Ω(k).
