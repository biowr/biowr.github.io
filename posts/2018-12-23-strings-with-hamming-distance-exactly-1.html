Strings with hamming distance exactly 111


Lin Yang asked me about the complexity for the following problem, which is the day 2 part 2 of the advent of code 2018. It is an elegant programming exercise, and also a clever algorithmic exercise. The problem can be summarized below.

Problem1

Given a set WWW of nnn length mmm strings. Decide if there are two of them that differs at exactly one position.

In other words, we want to find two strings in WWW with hamming distance 111.

The naive algorithm would have running time O(n2m)O(n^2m)O(n2m). The complexity of the problem have gathered a lot of attention a while ago, for example a post in dev.to, and on reddit. Some of them had a running time of O(nm2)O(nm^2)O(nm2) instead. Some require hashing to get the expected running time of O(mn)O(mn)O(mn). Here we are interested in an algorithm with worst case O(mn)O(mn)O(mn) time.

1 An O(mn)O(mn)O(mn) time algorithm

First, we define some equivalent classes on the strings in WWW.

  1. x≡iyx\equiv^i yx≡iy, if x[1..i−1]=y[1..i−1]x[1..i-1]=y[1..i-1]x[1..i−1]=y[1..i−1]. Namely, if the first i−1i-1i−1 elements of xxx and yyy match.
  2. x≡iyx\equiv_i yx≡i​y if x[i+1..m]=y[i+1..m]x[i+1..m]=y[i+1..m]x[i+1..m]=y[i+1..m]. Namely, if the last m−i+1m-i+1m−i+1 elements of xxx and yyy match.

The algorithm uses the following idea. For each iii, decide if there are any strings xxx and yyy such that differs in precisely position iii.

Lemma2

For distinct xxx and yyy, they differ only in position iii if and only if x≡iyx\equiv^i yx≡iy and x≡iyx\equiv_i yx≡i​y.

Let Pi\mathcal{P}_iPi​ and Si\mathcal{S}_iSi​ be the collection of equivalent classes of ≡i\equiv^i≡i and ≡i\equiv_i≡i​, respectively. We show a result related to the meet of partitions.

Lemma3

Let A\mathcal{A}A and B\mathcal{B}B be partitions of [n][n][n]. There is an O(n)O(n)O(n) time algorithm to test find the sets in {A∩B∣A∈A,B∈B}\set{ A\cap B | A\in \mathcal{A}, B\in \mathcal{B}}{A∩B∣A∈A,B∈B}.

Proof

Let A={A1,…,Ak}\mathcal{A}=\set{A_1,\ldots,A_k}A={A1​,…,Ak​} and B={B1,…,Bℓ}\mathcal{B} = \set{B_1,\ldots,B_\ell}B={B1​,…,Bℓ​}. Hence we are interested in find the largest set of elements such SSS such that for i,j∈Si,j\in Si,j∈S, Ii=IjI_i=I_jIi​=Ij​. The simplified problem can be solved in O(n)O(n)O(n) time. Indeed, the pair is just a base nnn number with 222 digits. We can apply radix sort with running time O(n)O(n)O(n) and group by the result.

As a corollary, consider A=Pi\mathcal{A}=\mathcal{P}_iA=Pi​ and B=Si\mathcal{B}=\mathcal{S}_iB=Si​, then we obtain the following lemma.

Lemma4

Given the collections Pi\mathcal{P}_iPi​ and Si\mathcal{S}_iSi​, there is an O(n)O(n)O(n) time algorithm to test if there are two strings x,y∈Wx,y\in Wx,y∈W that differs in precisely position iii.

Lemma5

Finding P1,…,Pm\mathcal{P}_1,\ldots,\mathcal{P}_mP1​,…,Pm​ and S1,…,Sm\mathcal{S}_1,\ldots,\mathcal{S}_mS1​,…,Sm​ can be done in O(mn)O(mn)O(mn) time.

Proof

To find the equivalent classes, build two tries for the strings. Trie TPT_\mathcal{P}TP​ for strings in WWW and trie TST_\mathcal{S}TS​ for the reversal of strings in WWW. Building the tries takes O(mn)O(mn)O(mn) time. Inspect the nodes at depth i−1i-1i−1 in TPT_PTP​ and nodes at depth m−i+1m-i+1m−i+1 in TST_STS​ to recover Pi\mathcal{P}_iPi​ and Si\mathcal{S}_iSi​ in O(n)O(n)O(n) time.

Theorem6

There is an O(mn)O(mn)O(mn) time algorithm that solves Problem 1.

Proof

Finding the sequence of equivalent classes takes O(mn)O(mn)O(mn) time by Lemma 4. For each iii, checking if there exists x,y∈Wx,y\in Wx,y∈W differs in precisely position iii takes O(n)O(n)O(n) time by Lemma 3. Since iii ranges from 111 to mmm, we obtain the final running time is O(mn)O(mn)O(mn).

2 Remarks

Ruosong Wang communicated another O(mn)O(mn)O(mn) solution. It is much easier to describe. Let ⋄\diamond⋄ be a symbol not in the alphabet. Build a generalized suffix tree over the set of strings S′={x⋄x∣x∈W}S'=\set{x\diamond x| x\in W}S′={x⋄x∣x∈W}. Traverse the suffix tree, up to level mmm, and output true if a path that contains ⋄\diamond⋄ was traversed, and can lead to more than 222 leaves. Indeed, this means the substring x⋄yx\diamond yx⋄y appears at least twice. Hence there are at least two strings of the form yaxyaxyax and ybxybxybx in WWW. This definitely hits the optimal running time, but implementing a generalized suffix tree is fairly hard.

We do assume the alphabet size is constant. If the alphabet size is σ\sigmaσ and ordered, then there is an extra factor of log⁡σ\log \sigmalogσ in building the tries. The the final running time will be O(mnlog⁡σ)O(mn\log \sigma)O(mnlogσ).

Problem 1 also reduces to finding the closest pair of elements by hamming metric [1]. It does not get us the desired running time though.
