Search in a sorted matrix with an oracle


Consider a infinite matrix MMM. Another way to think about it is a function f:N→N→Xf:\N\to \N\to Xf:N→N→X. A matrix MMM is sorted if Mi,j≤Mi,j+1M_{i,j}\leq M_{i,j+1}Mi,j​≤Mi,j+1​ and Mi,j≤Mi+1,jM_{i,j}\leq M_{i+1,j}Mi,j​≤Mi+1,j​.

Problem1

Given a sorted matrix MMM, and an oracle that takes λ\lambdaλ returns if a value is λ<λ∗\lambda <\lambda^*λ<λ∗ or λ≥λ∗\lambda \geq \lambda^*λ≥λ∗. Find the largest value no larger than λ∗\lambda^*λ∗.

Assuming there are at most kkk elements no larger than λ∗\lambda^*λ∗, and we know the smallest nnn and mmm such that Mi,j>λ∗M_{i,j}>\lambda^*Mi,j​>λ∗ if i>ni>ni>n or j>mj>mj>m. Also, let ttt be the smallest number such that Mt,t>λ∗M_{t,t}>\lambda^*Mt,t​>λ∗. One can see that t≤min⁡(n,m)t\leq \min(n,m)t≤min(n,m) and k=O(max⁡(n,m)2)k=O(\max(n,m)^2)k=O(max(n,m)2).

Let’s first consider the case when nnn and mmm is known and n≤mn\leq mn≤m. It is Leetcode 240. Search a 2D Matrix II. However, our problem is more general, because comparison with λ∗\lambda^*λ∗ can only be done through the oracle. Craig Gidney wrote about an optimal algorithm with O(nlog⁡mn)O(n\log \frac{m}{n})O(nlognm​) running time, matrix access algorithm. However, the oracle access is too large. There are times where the oracle access is slow. For example, when using it as a subroutine for finding a bottleneck kkk-link path. There is an algorithm with optimal running time and O(log⁡(nm))O(\log(nm))O(log(nm)) oracle access.

Let’s consider a special case, where n=m=2in=m=2^in=m=2i for some iii. This case was shown in [1]. For each submatrix, the two vertices on the opposite diagonal indicates the largest and smallest element in the submatrix. Hence each matrix can be represented by two numbers, indicate the maximum and minimum. These numbers are called the representative of the matrix. The idea is we keep two numbers λ1\lambda_1λ1​ and λ2\lambda_2λ2​, such that we know λ∗∈[λ1,λ2]\lambda^*\in [\lambda_1,\lambda_2]λ∗∈[λ1​,λ2​]. The algorithm keep partition the matrix into small matrices, updating λ1\lambda_1λ1​ and λ2\lambda_2λ2​, and discard matrices outside the range. We apply the following algorithm. Let RRR consists of the multiset of representatives of the matrix, and R′R'R′ be the representatives that lies inside [λ1,λ2][\lambda_1,\lambda_2][λ1​,λ2​]. We find λ\lambdaλ, the median of R′R'R′. Test if λ<λ∗\lambda<\lambda^*λ<λ∗. If so, then λ1\lambda_1λ1​ updates to λ\lambdaλ, otherwise λ2\lambda_2λ2​ updates to λ\lambdaλ. This step is done twice. Now, we split the matrices with more than one element into 444 equally sized matrices, and repeat the algorithm. Recall at all times, the matrices does not contain any element in [λ1,λ2][\lambda_1,\lambda_2][λ1​,λ2​] are discarded.

There is at most O(log⁡n)O(\log n)O(logn) iterations before range shrinks to a single element, hence at most O(log⁡n)O(\log n)O(logn) oracle calls. The difficulty is to show that the overall time is only O(n)O(n)O(n). Intuitively, in each iteration we quadruple the number of matrices, but we half it by two calls to the oracle. Therefore in log⁡n\log nlogn steps we obtain roughly 2log⁡n=O(n)2^{\log n}=O(n)2logn=O(n) matrices. However, at this point, the matrices are all singletons, and no more matrix can be created. We will only decrease the number of matrices by each oracle call. Careful reader can trace the whole argument in Lemma 2.1 of [1].

For the more general case, one can find the proof in [2]. Note the proof is for selection, but one can easily modify it to work for search.

Now, nnn and mmm is not known, but we can quickly using exponential search to find it. Indeed, we just have to apply exponential search in the first row and first column using the oracle. This gives us an extra O(log⁡n+log⁡m)=O(log⁡nm)O(\log n + \log m)=O(\log nm)O(logn+logm)=O(lognm) oracle calls.

Let kkk to be the number of elements no larger than λ∗\lambda^*λ∗. We can get running time relative to kkk. Use exponential search until we find the first iii such that M2i,2i>λ∗M_{2^i,2^i}>\lambda^*M2i,2i​>λ∗. So we can upper bound ttt. Then one can solve the problem with 222 matrices. One t×kt\times kt×k matrix and a k×tk\times tk×t matrix. The total running time is therefore O(log⁡k+tlog⁡k/t)=O(tlog⁡k)O(\log k+t\log k/t)=O(t\log k)O(logk+tlogk/t)=O(tlogk). In fact, we get O(log⁡k)O(\log k)O(logk) oracle calls and O(tlog⁡k)O(t\log k)O(tlogk) running time. Here we can take ttt to be k\sqrt{k}k​, and obtain O(klog⁡k)O(\sqrt{k}\log k)O(k​logk) time.

Note if we relax on number of oracle calls. I know how to get a O(k)O(\sqrt{k})O(k​) running time.

Theorem2

Given λ∗\lambda^*λ∗ and a n×mn\times mn×m sorted matrix such that the iiith row has kik_iki​ elements no larger than xxx. Let k=∑ikik=\sum_{i} k_ik=∑i​ki​. We can find λ∗\lambda^*λ∗ in O(∑ilog⁡(ki+1−ki+1))=O(nlog⁡kn2)O(\sum_{i} \log (k_{i+1}-k_i+1) ) = O(n \log \frac{k}{n^2})O(∑i​log(ki+1​−ki​+1))=O(nlogn2k​) time.

The idea is simple, we do exponential search on each row to find the largest element no larger than λ∗\lambda^*λ∗, but we reuse information from the previous row. This gives us the running time O(∑ilog⁡(ki+1−ki+1))O(\sum_{i} \log (k_{i+1}-k_i+1) )O(∑i​log(ki+1​−ki​+1)). The main difficulty is to show why is is O(nlog⁡kn2)O(n \log \frac{k}{n^2})O(nlogn2k​).

Lemma3

If ∑i=1nki=k\sum_{i=1}^n k_i=k∑i=1n​ki​=k, then ∑i=1nlog⁡(ki+1−ki+1)=O(nlog⁡k/n2)\sum_{i=1}^n \log (k_{i+1}-k_i+1)=O(n\log k/n^2)∑i=1n​log(ki+1​−ki​+1)=O(nlogk/n2).

Once we show that, we can use the theorem to obtain O(k)O(\sqrt{k})O(k​) running time.

In fact, we can also get a very simple algorithm with running time O(hlog⁡(k/h2))O(h \log(k/h^2))O(hlog(k/h2)), where hhh is the number of stairs in the staircase shape. However, it also uses O(hlog⁡(k/h2)O(h\log(k/h^2)O(hlog(k/h2) oracle calls. The idea is to use exponential search to find the boundary of the staircase, but we switch between boundaries: horizontal then vertical. It is open if we can obtain O(hlog⁡(k/h2))O(h\log(k/h^2))O(hlog(k/h2)) running time and O(log⁡k)O(\log k)O(logk) oracle calls.

1 Remark

There is an alternative algorithm which can be found in [3]. The alternative algorithm is quite close to a post about selection in a sorted matrix.

The careful reader might observe the known search algorithms follow the exact same structure as algorithms for selection. Indeed, we are doing selection but we do not know the rank of the element. Intuitively, many selection algorithm, the rank is only used to remove the correct set of candidates. Hence this suggest one can modify the algorithm to use the oracle call in place of the rank.
