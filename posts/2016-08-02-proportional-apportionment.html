Processor distribution and proportional apportionment


I saw an interview problem about assigning identical processors to embarrassingly parallel jobs. The running time of a job equals the running time on a single processor divided by the number of processors. We are interested in minimizing the maximum running time. Formally, we get the following problem.

Problem1

Given positive reals a1,…,ana_1,\ldots,a_na1​,…,an​ and positive integer kkk, find non-negative integers x1,…,xnx_1,\ldots,x_nx1​,…,xn​, such that ∑ixi≤k\sum_{i} x_i \leq k∑i​xi​≤k and θ=max⁡iai/xi\theta = \max_{i} a_i/x_iθ=maxi​ai​/xi​ is minimized.

If there is no integral requirement on xix_ixi​’s, then the problem is easy. Let A=∑iaiA=\sum_{i} a_iA=∑i​ai​. There is a closed solution of xi=kaiAx_i = k \frac{a_i}{A}xi​=kAai​​, and θ=A/k\theta = A / kθ=A/k.

Otherwise, it is easy to check if θ′>0\theta'>0θ′>0 is a feasible solution. θ′\theta'θ′ is feasible iff ∑i⌈ai/θ′⌉≤k\sum_{i} \ceil{a_i/\theta'} \leq k∑i​⌈ai​/θ′⌉≤k. Therefore one can apply binary search, and get the result in O(nlog⁡k)O(n\log k)O(nlogk) time.

One can also get a O(nlog⁡n)O(n\log n)O(nlogn) time algorithm. First compute yi=⌈kaiA⌉y_i = \ceil{k \frac{a_i}{A}}yi​=⌈kAai​​⌉. Greedily find a iii such that ai/yia_i/y_iai​/yi​ is maximized, and decrease yiy_iyi​ by 111. We stop when we have ∑iyi=k\sum_{i} y_i=k∑i​yi​=k. This takes O(log⁡n)O(\log n)O(logn) per operation using a binary search tree.

Linear time algorithm also exists. It is connected to proportional apportionment. This is the problem of finding the smallest λ\lambdaλ, such that ∑i⌈λai⌉=k\sum_{i} \ceil{\lambda a_i} = k∑i​⌈λai​⌉=k. Cheng and Eppstein found a O(n)O(n)O(n) time algorithm [1]. Reitzig and Wild found a simpler algorithm later [2].

There is a similar interview problem. Given nnn points on the real line, add kkk more points, such that it minimizes the maximum length between adjacent points. The problem is the same as the following one.

Problem2

Given positive a1,…,ana_1,\ldots,a_na1​,…,an​ and positive integer kkk, find non-negative integers x1,…,xnx_1,\ldots,x_nx1​,…,xn​, such that ∑ixi≤k\sum_{i} x_i \leq k∑i​xi​≤k and θ=max⁡iai/(xi+1)\theta = \max_{i} a_i/(x_i+1)θ=maxi​ai​/(xi​+1) is minimized.

The linear time algorithm for proportional apportionment should also work for the above problem. It is interesting how much can we change the problem before the linear time algorithm no longer works.
