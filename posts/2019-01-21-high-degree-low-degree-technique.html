The high-degree low-degree technique and arboricity


In this piece we demonstrate the high-degree low-degree technique in graphs. Often, we obtain running times that depends on the individual degrees of the vertices. If the graph has only low degree vertices, then a faster algorithm exists. For graph with only large degrees, then it is dense, and can often be handled in another way.

We will also use the information of arboricity. Mainly, there are a few useful statements.

Theorem1

For a graph G=(V,E)G=(V,E)G=(V,E) with arboricity α\alphaα, we have ∑uv∈Emin⁡(deg⁡(u),deg⁡(v))≤2αm \sum_{uv\in E} \min(\deg(u),\deg(v)) \leq 2\alpha m uv∈E∑​min(deg(u),deg(v))≤2αm

Often, using the arboricity, we can obtain the same complexity algorithm without high-degree low-degree technique. Note the arboricity is O(m)O(\sqrt{m})O(m​). The application of arboricity are from [1].

Some of the algorithms described can be speedup by using matrix multiplication, or faster combinatorial boolean matrix multiplication. We avoid them for simplicity of exposition.

Theorem3

There is a dominating set incident to O(nnlog⁡n)O(n\sqrt{n\log n})O(nnlogn​) edges.

Proof

Fix a δ\deltaδ. We repeatedly removing vertices with degree no more than δ\deltaδ from the graph, and add it into a set DDD. The total degree of DDD is at most nδn\deltanδ. Now the remaining vertices has degree at least δ\deltaδ. Using the set cover theorem, and let the distribution to be the uniform distribution. If all elements are covered at least by ε\eε fraction of the set, then we obtain a set cover of size O(log⁡uε)O(\frac{\log u}{\e})O(εlogu​). Now, let the sets N(v)N(v)N(v) for each vvv. One can show the above result is almost optimal, as there exists graphs where every dominating set incidents Ω(n3/2)\Omega(n^{3/2})Ω(n3/2) edges. The same bound holds for weakly connected dominating set, that is a dominating set DDD such that the edges incident to DDD forms a connected graph. The stronger modification of this result was used in deciding the 444-connectivity of a matroid [2].

2 Finding small subgraphs

2.1 Finding a triangle

A triangle is 333 vertices pairwise adjacent to each other, another name for K3K_3K3​.

Theorem4

There is a O(mΔ)O(m\Delta)O(mΔ) time algorithm to decide if the graph has a triangle, where Δ\DeltaΔ is the maximum degree.

Proof

Indeed, for each vertex vvv, we consider its neighbors, see if any is adjacent to each other. We then delete vvv. The algorithm takes O(∑vdeg⁡2(v))=O(mΔ)O(\sum_{v} \deg^2(v)) = O(m\Delta)O(∑v​deg2(v))=O(mΔ) time.

Theorem5

There is a O(n3)O(n^3)O(n3) time algorithm to decide if the graph has a triangle.

Proof

The naive algorithm, for each 333 vertices, we decide if it forms a triangle.

Theorem6

There is a O(m3/2)O(m^{3/2})O(m3/2) time algorithm to decide if the graph has a triangle.

Proof

Let ttt be a parameter we will find later. Apply the above algorithm by picking the vertex with the smallest degree, until the next vertex has degree at least ttt. It will use at most O(mt)O(mt)O(mt) time. Now, for the remaining graph, it is clear the maximum degree is at least ttt. Note, there can be at most n/tn/tn/t vertices. We use the O(n3)O(n^3)O(n3) time algorithm. The final running time is O(mt+(m/t)3)O(mt+(m/t)^3)O(mt+(m/t)3). Set t=mt=\sqrt{m}t=m​ and we are done.

2.2 A motivating problem

Let S1,…,SnS_1,\ldots,S_nS1​,…,Sn​ be sets with total of mmm elements. How quickly can we find two distinct iii and jjj such that ∣Si∩Sj∣≥ℓ|S_i\cap S_j|\geq \ell∣Si​∩Sj​∣≥ℓ? This problem can be shown to be equivalent to finding a colored K2,ℓK_{2,\ell}K2,ℓ​ in a bipartite graph. That is, for input bipartite graph G=(A,B,E)G=(A,B,E)G=(A,B,E). 2.3 Finding a C4C_4C4​ in bipartite graphs

This section we use technique that follows from [3]. Although we are into finding C4C_4C4​, but some theorems are more general for K2,ℓK_{2,\ell}K2,ℓ​, so we will state them too. Note finding a colored K2,2K_{2,2}K2,2​ and finding a K2,2K_{2,2}K2,2​ is the same problem due to symmetry.

Let v1,…,vnv_1,\ldots,v_nv1​,…,vn​ be an ordering such that deg⁡(vi)≥deg⁡(vj)\deg(v_i)\geq \deg(v_j)deg(vi​)≥deg(vj​). Here Ni(v)N_i(v)Ni​(v) is the set of neighbors of vvv in G[{vi,…,vn}]G[\set{v_i,\ldots,v_n}]G[{vi​,…,vn​}]. Here we show an algorithm solves the above problem when the arboricity is small.

The algorithm is as follows [1]. Take vvv, we consider each neighbor uuu. Maintain a set SwS_wSw​ for each vertex www distance 222 from vvv. Add uuu into each of uuu’s neighbor in www. SwS_wSw​ would give us information of N(v)∩N(w)N(v)\cap N(w)N(v)∩N(w). We delete vvv and keep going. It is known the algorithm takes O(α(G)m)O(\alpha(G)m)O(α(G)m) time. This allows us to compute C4C_4C4​ in the same time. Hence we directly obtain O(m3/2)O(m^{3/2})O(m3/2) running time. However, we show something better is possible if we are not interested in finding all C4C_4C4​, but find any C4C_4C4​. We also need the following theorem.

2.4 Finding a colored K2,3K_{2,3}K2,3​ in bipartite graphs

For finding K2,ℓK_{2,\ell}K2,ℓ​, the low arboricity algorithm for C4C_4C4​ works here. The O(α(G)m)O(\alpha(G)m)O(α(G)m) algorithm is still fine. It’s not hard to generalize and show a O(ℓ1/3m4/3)O(\ell^{1/3}m^{4/3})O(ℓ1/3m4/3) running time algorithm.

However, in order to solve the motivating problem. We need a colored K2,ℓK_{2,\ell}K2,ℓ​.
